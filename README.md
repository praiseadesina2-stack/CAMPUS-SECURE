CampusSecure – Lost & Found System
Still in the development Phase!!!
=========================
CampusSecure is a simple web-based Lost and Found system designed for university campuses. I built this project to demonstrate how technology can help students and staff recover lost items faster and more securely.

The platform allows users to:

Report items they have found on campus

Search for lost items

View recently reported items

Submit claims for ownership verification

This project simulates a pilot deployment at Covenant University, where campus security and students can collaborate to return items to their rightful owners.

Project Purpose
====================

The goal of this project is to create a centralized digital system for managing lost and found items on campus.

In many universities, lost items are handled informally through security offices or word of mouth. CampusSecure improves this process by providing:

A central database of found items

Quick search functionality

Secure item claiming process

User-friendly dashboard

Features 
============= 
1. Dashboard

The dashboard displays:

Total number of reported items

Number of items returned

Average return time

Recently reported found items

This gives users a quick overview of activity within the system.

2. Report Found Item

Users can report items they have found by submitting:

Item title

Description

Category

Photo upload

Once submitted, the item appears immediately on the dashboard.

3. Search Lost Items

Users can search the database using keywords such as:

Item name

Category

Location

The system dynamically filters results in real time.

4. Item Details Modal

Clicking an item opens a modal that shows:

Item image

Description

Location where it was found

Who reported it

From here, users can attempt to claim the item.

5. Claim Verification

To claim an item, the user must:

Answer a verification question

Upload proof of ownership

Security staff can then review the claim before releasing the item.

6. Notifications

The notifications section is designed to alert users when:

A matching item is found

A claim is approved or rejected

(This is currently simulated in the demo.)

7. User Profile

The profile page shows:

User name

Role

Points earned

Points are intended to reward users who report found items honestly.

Example Use Case
==============

A student finds a USB drive in the lecture hall

They open CampusSecure and report the found item

Another student searches for "USB"

They recognize the item and submit a claim

Security verifies ownership and returns the item
